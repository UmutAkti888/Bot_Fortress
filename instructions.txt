## What we are building

A local web application called **BotHub** — a personal dashboard to run and manage bots from a browser.

The first bot is an **ArXiv Paper Fetcher** that:
- Accepts keywords from the user (e.g. "SLAM", "DBSCAN", "mobile robotics")
- Searches the ArXiv API (free, no key required)
- Downloads matching PDFs to a local `/papers` folder
- Saves paper metadata (title, authors, abstract, link, date) to `results.json`

More bots will be added later. The architecture must make it easy to plug in new bots.

## ArXiv API details

- Base URL: `http://export.arxiv.org/api/query`
- Example query: `http://export.arxiv.org/api/query?search_query=all:SLAM+AND+all:mapping&max_results=10`
- Response format: Atom XML feed (use `feedparser` to parse)
- PDF link format: replace `abs` with `pdf` in the entry link, e.g.:
  `https://arxiv.org/abs/2301.12345` → `https://arxiv.org/pdf/2301.12345`
- Relevant ArXiv category for robotics: `cs.RO`
